## 🚨 КРИТИЧЕСКИЕ ПРОБЛЕМЫ БЕЗОПАСНОСТИ

### 1. ❌ Захардкоженные WooCommerce API ключи
**Критичность:** 🔴 КРИТИЧЕСКАЯ  
**Файлы:**
- `src/app/services/api.ts` (строки 31-32)
- `src/app/services/publicApi.ts` (строки 12-13)
- `src/app/services/wooCommerceApi.ts` (строки 15-16)
- `src/components/AuthTest.tsx` (строки 60-61)

**Проблема:** WooCommerce Consumer Key и Consumer Secret захардкожены в коде и будут видны всем в продакшн-сборке.

**Решение:** 
- Вынести в переменные окружения `VITE_WC_CONSUMER_KEY` и `VITE_WC_CONSUMER_SECRET`
- Удалить из кода все хардкод значения
- Добавить проверку наличия переменных окружения

**Риск:** Любой может получить доступ к вашему WooCommerce API и управлять заказами, товарами, клиентами.

## ⚠️ ВАЖНЫЕ ПРОБЛЕМЫ

### 2. Множество console.log в продакшн-коде
**Критичность:** 🟡 СРЕДНЯЯ  
**Проблема:** Более 240 вызовов `console.log/error/warn` в коде, которые будут видны в консоли браузера в продакшне.

**Решение:**
- Обернуть все console.log в проверку `import.meta.env.DEV`
- Или использовать утилиту для логирования только в dev режиме
- Удалить отладочные логи из критических путей

**Файлы с большим количеством логов:**
- `src/app/services/api.ts` (~50 вызовов)
- `src/app/services/apiConfig.ts` (~15 вызовов)
- `src/app/services/authService.ts` (~20 вызовов)
- `src/components/Dashboard/EditProductModal.tsx` (~15 вызовов)
- `src/components/Dashboard/AddProductModal.tsx` (~10 вызовов)

---

### 3. Отсутствие .env.example файла
**Критичность:** 🟡 СРЕДНЯЯ  
**Проблема:** Нет примера файла с переменными окружения для других разработчиков.

**Решение:** Создать `.env.example` с описанием всех необходимых переменных.

---

### 4. .gitignore не содержит явного исключения .env файлов
**Критичность:** 🟡 СРЕДНЯЯ  
**Проблема:** Хотя `*.local` исключен, лучше явно указать `.env*` для ясности.

**Решение:** Добавить `.env*` в `.gitignore`.

---

### 5. Отсутствие проверки env переменных в продакшне
**Критичность:** 🟡 СРЕДНЯЯ  
**Проблема:** В `apiConfig.ts` есть проверка, но она только выводит ошибку в консоль. В продакшне это может быть незаметно.

**Решение:** Добавить более строгую проверку с понятными сообщениями об ошибках для пользователей.

---

### 6. Тестовый компонент AuthTest.tsx в продакшне
**Критичность:** 🟡 СРЕДНЯЯ  
**Проблема:** Компонент `AuthTest.tsx` содержит тестовый код и захардкоженные ключи.

**Решение:** 
- Удалить из продакшн-сборки или условно рендерить только в dev режиме
- Удалить захардкоженные ключи

## ⚙️ Минимальная настройка (5 минут)

### Шаг 1: Обновить адреса ресторана

**Откройте:** `src/components/Checkout.tsx`

**Найдите (строки 10-11):**
```typescript
const RESTAURANT_ADDRESS = 'ул. Абдымомунова, 265, Бишкек 720040, Кыргызстан';
const RESTAURANT_PHONE = '+996 (312) 62-55-55';
```

**Замените на реальные данные вашего ресторана:**
```typescript
const RESTAURANT_ADDRESS = 'ваш_адрес_здесь';
const RESTAURANT_PHONE = 'ваш_телефон_здесь';
```

### Шаг 2: Обновить плагин WordPress

**Скопируйте файл:**
```
wordpress-plugins/order-type-display.php
```

**В WordPress:**
```
wp-content/plugins/order-type-display.php
```

### Шаг 3: Активировать плагин

```
WordPress админ-панель → Плагины → Order Type Display → Активировать
```

## 🧪 Тестирование

### Тест 1: Создание заказа с доставкой

1. Откройте сайт
2. Добавьте товар в корзину
3. Нажмите "Оформить заказ"
4. Выберите "🛍️ Доставка"
5. Введите адрес доставки
6. Подтвердите заказ
7. ✅ Заказ должен быть создан

**Проверка в админ-панели:**
- Откройте заказ в WooCommerce
- Должна быть видна строка "🚚 Тип заказа: Доставка"

### Тест 2: Создание заказа с самовывозом

1. Добавьте товар в корзину
2. Нажмите "Оформить заказ"
3. Выберите "📦 Самовывоз"
4. Проверьте, что:
   - ✅ Адрес доставки исчезает
   - ✅ Показывается адрес ресторана
5. Подтвердите заказ
6. ✅ Заказ должен быть создан

**Проверка в админ-панели:**
- Откройте заказ в WooCommerce
- Должна быть видна строка "📦 Тип заказа: Самовывоз"
- Адрес должен быть адресом ресторана

## 🔍 Проверка в браузере

Откройте консоль (F12) и смотрите на сетевые запросы:

### При создании заказа должно быть:

```json
{
  "status": "on-hold",
  "customer_id": 0,
  "billing": {
    "first_name": "Имя",
    "address_1": "Адрес или адрес ресторана",
    "phone": "+996712345678",
    "email": "customer_1234567890@example.com"
  },
  "line_items": [...],
  "meta_data": [
    {
      "key": "order_type",
      "value": "pickup"  // или "delivery"
    }
  ]
}
```

---

## 📊 Данные в базе

В WordPress база данных (`wp_postmeta`):

```
post_id: 123
meta_key: order_type
meta_value: "pickup"
```

Проверить можно через phpMyAdmin или WordPress CLI:
```bash
wp post meta get <order_id> order_type
```

## 🐛 Если что-то не работает

### Проблема: Адрес не отправляется
1. Откройте F12 → Network
2. При создании заказа найдите запрос POST
3. Проверьте payload → должна быть `meta_data`
4. Если её нет - обновите страницу и попробуйте снова

### Проблема: Тип не видно в админке
1. Проверьте, активирован ли плагин:
   - WordPress → Плагины → Order Type Display
2. Обновите страницу заказа (Ctrl+Shift+R)
3. Проверьте консоль браузера на ошибки

### Проблема: Письмо не отправляется
1. Проверьте логи WooCommerce: `wp-content/uploads/wc-logs/`
2. Убедитесь, что SMTP настроен правильно

## ⚠️ ВАЖНЫЕ ШАГИ ПЕРЕД ДЕПЛОЕМ

### 1. Создайте .env файл для локальной разработки

Скопируйте `.env.example` в `.env` и заполните реальными значениями:

```bash
cp .env.example .env
```

Заполните `.env` файл:
```env
VITE_API_BASE_URL=https://your-wordpress-site.com/wp-json/
VITE_WC_CONSUMER_KEY=ck_your_real_key_here
VITE_WC_CONSUMER_SECRET=cs_your_real_secret_here
VITE_WP_USERNAME=your_username
VITE_WP_APP_PASSWORD=xxxx xxxx xxxx xxxx xxxx xxxx
```

### 2. Настройте переменные окружения на хостинге

#### Для Vercel:
1. Перейдите в настройки проекта
2. Settings > Environment Variables
3. Добавьте все переменные из `.env.example`
4. Убедитесь, что они установлены для Production

#### Для Netlify:
1. Перейдите в настройки сайта
2. Site settings > Environment variables
3. Добавьте все переменные из `.env.example`
4. Убедитесь, что они установлены для Production

### 3. Проверьте сборку

```bash
npm run build
```

Убедитесь, что:
- ✅ Сборка проходит без ошибок
- ✅ Размер бандла разумный (проверьте `dist/` папку)
- ✅ Нет предупреждений о отсутствующих переменных окружения

### 4. Протестируйте локально продакшн-сборку

```bash
npm run build
npm run preview
```

Проверьте:
- ✅ Приложение запускается
- ✅ API запросы работают
- ✅ Авторизация работает
- ✅ Загрузка изображений работает (если используется)

### 5. Проверьте безопасность

- [ ] Убедитесь, что `.env` файл не попал в git (проверьте `git status`)

- [ ] Убедитесь, что все переменные окружения установлены на хостинге
